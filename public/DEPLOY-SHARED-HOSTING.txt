Billo in a subfolder: https://www.webspace.ng/billo/login

HOME WORKS BUT /billo/login → HOSTINGER 404
  Almost always one of:
  1) File public_html/billo/.htaccess is missing (FTP often hides “dotfiles” — enable “show hidden”).
  2) Old .htaccess on server (must use RewriteRule .* index.php, not RewriteRule ^ index.php).
  3) Folder name in URL ≠ folder on disk (Linux: /billo ≠ /Bilo). config app.base_path and RewriteBase must match the real URL segment.
  4) Files are in public_html/billo/public/ — then either move contents up into billo/ OR use URL .../billo/public/login and set base_path /billo/public (not recommended).

WHERE FILES GO
  public_html/billo/
    index.php
    .htaccess          ← required; same folder as index.php
    billo-root.php     ← only if you use split deploy (see billo-root.php.example)
    assets/
    routing-check.php

  config app.base_path: '/billo'  (same spelling/case as in the URL)

HTACCESS FIX ON SERVER
  1) Re-upload public/.htaccess from this repo into public_html/billo/
  2) In hPanel File Manager: enable “Show Hidden Files” and confirm .htaccess is there.
  3) If login still 404: edit .htaccess — uncomment ONE line:
       RewriteBase /billo/
     Use the exact path segment you see in the browser (e.g. /Bilo/ if folder is capital B).

FALLBACKRESOURCE
  This stack also sets FallbackResource index.php so many Hostinger/LiteSpeed setups route /login even when RewriteBase was wrong.

VERIFY
  https://www.webspace.ng/billo/routing-check.php  → plain text
  https://www.webspace.ng/billo/login             → Billo login form

WWW
  Set config app.url to https://www.webspace.ng (with www) if that is what you use.
