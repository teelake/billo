Billo in a subfolder: https://www.webspace.ng/billo/login

HOME WORKS BUT /billo/login → HOSTINGER 404
  Almost always one of:
  1) File public_html/billo/.htaccess is missing (FTP often hides “dotfiles” — enable “show hidden”).
  2) Old .htaccess on server (must use RewriteRule .* index.php, not RewriteRule ^ index.php).
  3) Folder name in URL ≠ folder on disk (Linux: /billo ≠ /Bilo). config app.base_path and RewriteBase must match the real URL segment.
  4) Files are in public_html/billo/public/ — then either move contents up into billo/ OR use URL .../billo/public/login and set base_path /billo/public (not recommended).

TWO VALID LAYOUTS

  (1) FULL PROJECT under public_html/billo/ (app, config, public, vendor, …)
      → Put the project ROOT .htaccess here: public_html/billo/.htaccess
      → Front controller: public/index.php (do not rely on index.php in billo root)
      → This is the usual case when you upload the whole Git repo into billo/

  (2) ONLY /public contents under public_html/billo/
      → Use public/.htaccess from the repo (not the root .htaccess)
      → index.php sits next to that .htaccess
      → Use billo-root.php if app/config live outside web root

WHERE FILES GO — layout (1)
  public_html/billo/
    .htaccess          ← copy from project ROOT (billo/.htaccess), NOT from public/
    app/
    config/
    public/
      index.php
      assets/
    vendor/

  Optional in public/: billo-root.php (split deploy), routing-check.php

  config app.base_path: '/billo'  (same spelling/case as in the URL)

HTACCESS FIX ON SERVER
  If layout (1) — full project in public_html/billo/:
  1) Upload the project ROOT .htaccess (repo file billo/.htaccess) to public_html/billo/.htaccess
     — NOT public/.htaccess (that file is only for layout 2).
  2) In hPanel File Manager: enable “Show Hidden Files” and confirm .htaccess is there.
  3) If /billo/login still 404: edit RewriteBase in that file to match your URL (case-sensitive).

FALLBACKRESOURCE
  The root .htaccess sets FallbackResource public/index.php so deep URLs still hit the front controller when rewrites behave oddly.

VERIFY
  https://www.webspace.ng/billo/routing-check.php  → plain text
  https://www.webspace.ng/billo/login             → Billo login form

WWW
  Set config app.url to https://www.webspace.ng (with www) if that is what you use.

PLATFORM OPERATOR (/system)
  There is no separate “system login”. Use the normal /login, then open /system.
  To create your owner account (internal org + is_system_admin), SSH or local PHP with DB configured:
    cd .../billo
    php scripts/bootstrap-platform-admin.php your@email.com
  A random password is printed once. Or set it explicitly:
    php scripts/bootstrap-platform-admin.php your@email.com --password='YourStr0ng!Pass'
  Re-run safely: same email gets system admin + org link; add --password to rotate.
